## AI Summary Generation

Each audit result includes a personalized ~100-word summary paragraph generated by Anthropic's Claude API. This summary appears at the top of the results page and in the email report, providing a human-readable narrative of the audit findings.

### System Prompt

```
You are a concise financial advisor specializing in AI tool spending optimization for tech teams. Given an audit of a team's AI tool subscriptions, write a personalized ~100-word summary paragraph that:
1. Acknowledges what they're doing well (if anything)
2. Highlights the biggest savings opportunity with specific numbers
3. Gives one actionable next step
4. Uses a professional but approachable tone — like a smart colleague, not a robot

Do NOT use bullet points, headers, or markdown. Write a single flowing paragraph.
Do NOT mention Credex or any consulting service.
Do NOT make up numbers — only reference the data provided.
```

### User Prompt Template

```
Audit Summary for a {{teamSize}}-person team (primary use case: {{useCase}}):

Current monthly spend: ${{totalCurrentSpend}}
Recommended monthly spend: ${{totalRecommendedSpend}}
Total potential savings: ${{totalMonthlySavings}}/month (${{totalAnnualSavings}}/year)

Tool-by-tool breakdown:
{{#each toolResults}}
- {{tool}} ({{currentPlan}}): ${{currentSpend}}/mo → {{recommendedAction}}: {{reasoning}}
{{/each}}

Write a ~100-word personalized summary paragraph.
```

The `{{handlebars}}` syntax above is illustrative — the actual implementation uses TypeScript template literals to interpolate values from the `AuditResult` object.

### Example Output

Given a 12-person team spending $1,140/mo across Cursor Teams, GitHub Copilot Business, and Claude Team Standard:

> Your team has made a smart choice with Cursor Teams for your primary IDE — at $40/seat it's competitive and the agent capabilities are strong for a full-stack team. The biggest opportunity here is the $228/month you're spending on GitHub Copilot Business alongside Cursor; since both tools provide AI code completion, you're paying for significant functional overlap. Dropping Copilot and consolidating on Cursor alone would save your team $2,736 annually. I'd recommend running a two-week trial with your team using only Cursor to validate that no critical Copilot-specific workflows are lost before canceling.

## Design Decisions

### Why a single paragraph (no markdown)?

The AI summary is rendered in a styled `<p>` tag within a card component on the results page. Bullet points, headers, or markdown formatting would require a markdown renderer and create inconsistent visual styling. A single flowing paragraph:
- Renders cleanly in any context (web, email, OG image)
- Feels more personal and human than a bulleted list
- Is easier to constrain to ~100 words

### Why "smart colleague" tone?

The target user is a tech lead or engineering manager evaluating their team's AI tool spend. They want **actionable insight**, not corporate jargon or AI-sounding platitudes. The "smart colleague" instruction produces output that reads like advice from a knowledgeable peer — direct, specific, and unpretentious.

### Why prohibit mentioning Credex?

The AI summary should feel like **objective analysis**, not a sales pitch. Brand promotion is handled by the surrounding UI (header, CTA buttons, email footer). If the AI mentions Credex, it undermines trust in the recommendation's objectivity. The separation of concerns is:
- **AI summary** → neutral, data-driven analysis
- **UI layer** → branding, CTAs, lead capture

### Why prohibit making up numbers?

Large language models can hallucinate plausible-sounding numbers. The instruction "only reference the data provided" anchors Claude's output to the actual audit data passed in the user prompt. Combined with providing all relevant numbers in the prompt itself, this minimizes the risk of fabricated statistics appearing in the summary.

### Why the final instruction "Write a ~100-word personalized summary paragraph"?

Repeating the core instruction at the end of the user prompt (after the data) serves as a final reminder and helps prevent the model from defaulting to other output formats. This is a standard prompt engineering technique — the model pays strong attention to both the beginning and end of prompts.

## Template Fallback

If the Anthropic API call fails (timeout, rate limit, 500 error, network issue), the system falls back to a deterministic template that produces a comparable paragraph using the same audit data.

### Fallback Logic

```typescript
function generateFallbackSummary(result: AuditResult): string {
  const { teamSize, totalMonthlySavings, totalAnnualSavings, toolResults } = result;
  
  // Find the tool with the largest savings opportunity
  const biggestSaving = toolResults
    .filter(t => t.monthlySavings > 0)
    .sort((a, b) => b.monthlySavings - a.monthlySavings)[0];
  
  // Find tools that are well-optimized (no change recommended)
  const wellOptimized = toolResults
    .filter(t => t.recommendedAction === 'keep');
  
  let summary = '';
  
  if (wellOptimized.length > 0) {
    summary += `Your ${teamSize}-person team is making good use of ${wellOptimized.map(t => t.tool).join(' and ')}, which ${wellOptimized.length === 1 ? 'is' : 'are'} well-suited for your needs. `;
  }
  
  if (biggestSaving) {
    summary += `The biggest opportunity is with ${biggestSaving.tool}: ${biggestSaving.reasoning}. `;
    summary += `This single change could save $${biggestSaving.monthlySavings}/month. `;
  }
  
  summary += `Overall, optimizing your AI tool stack could save $${totalMonthlySavings}/month — that's $${totalAnnualSavings} annually. `;
  summary += `We recommend starting with the highest-impact change and evaluating the rest over the next billing cycle.`;
  
  return summary;
}
```

### Why a template fallback instead of no summary?

The AI summary is a **core feature** of the results page — it's the first thing users read and the most frequently shared element. If it were simply omitted on API failure, the results page would feel incomplete. The template fallback ensures:
- Every audit always has a summary
- The feature degrades gracefully (slightly less personalized, but still useful)
- Users never see an error state or empty section

## Model Configuration

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| **Model** | `claude-haiku-4-5-20250514` | Cost-optimized for short-form generation; summary quality is comparable to Sonnet for this task |
| **Max Tokens** | `300` | ~100 words ≈ ~150 tokens; 300 provides headroom without allowing runaway generation |
| **Temperature** | `0.7` | Slight creativity for natural-sounding prose while keeping output grounded in provided data |
| **Timeout** | `10 seconds` | Aggressive timeout ensures the audit API endpoint responds quickly; triggers fallback on slow responses |

### Cost Per Audit

With Claude Haiku 4.5 at $1.00/1M input tokens and $5.00/1M output tokens:

- **Average input**: ~400 tokens (system prompt + user prompt with data) → $0.0004
- **Average output**: ~150 tokens (summary paragraph) → $0.00075
- **Total per audit**: ~$0.001 (one-tenth of a cent)
- **At 1,000 audits/month**: ~$1.00/month

This makes AI summary generation essentially free at MVP scale.
